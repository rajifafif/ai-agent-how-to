# Testing Standard

Every meaningful change needs appropriate verification.

## Possible Layers

- unit tests;
- integration tests;
- API tests;
- database tests;
- UI/end-to-end tests;
- regression tests;
- linting;
- type checks;
- static analysis;
- security checks.

## Evidence Rule

Do not claim a test passed unless it was executed successfully. If not run, state why.

## Repository Rule

Use the project commands documented in `AGENTS.md`, package files, CI config, or scripts. Do not invent commands.
