# Standards

Company-wide rules that apply across projects.

Read these as the source of truth for organization-level behavior. Project repositories may
add stricter rules in `AGENTS.md` or project docs, but should not silently weaken these
standards.

## Core Standards

| Standard | Purpose |
| --- | --- |
| `ai-usage-standard.md` | How developers, QA, and AI agents should work together. |
| `ai-ready-repository-standard.md` | Minimum files and guardrails for AI-ready repositories. |
| `secure-coding-standard.md` | Security expectations and OWASP-aligned review areas. |
| `testing-standard.md` | Minimum testing and verification expectations. |
| `documentation-standard.md` | Documentation governance and source-of-truth rules. |
| `commit-standard.md` | Logical commit rules and safety checks. |
| `pull-request-standard.md` | PR content, evidence, QA handover, and risk expectations. |
| `human-review-standard.md` | Actions requiring explicit human approval. |

## Usage

- Organization standards live here.
- Project-specific facts live in project `AGENTS.md` and project docs.
- Task-specific context comes from issues, plans, bug reports, or user instructions.
