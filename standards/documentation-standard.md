# Documentation Standard

Documentation should be small, accurate, current, non-duplicated, and routed from a clear entry point.

## Source of Truth Pattern

```text
AGENTS.md -> AI operating rules
README.md -> project overview
docs/ARCHITECTURE.md -> architecture
docs/DEVELOPMENT.md -> setup/development
docs/TESTING.md -> verification
docs/SECURITY.md -> project-specific security
```

## Avoid

- duplicate instructions;
- stale architecture notes;
- conflicting prompts;
- long generic essays;
- undocumented temporary rules.

Update only documentation affected by the change.
