# Pull Request Standard

Developers should be able to ask: `Create a pull request from this branch`.

## Required PR Sections

- Summary
- Problem
- Solution
- Scope
- Affected modules
- Testing performed
- Test evidence
- Security considerations
- OWASP review
- Database/migration changes
- Configuration changes
- API changes
- Documentation updated
- Risks
- Rollback plan
- QA handover
- Checklist

## Evidence Rule

The AI agent must inspect actual branch diff and commits before preparing PR content. It must not claim tests passed unless observed.
