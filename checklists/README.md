# Checklists

Small, copyable gates for real project work.

Checklists are intentionally shorter than standards. Use them during reviews, PR prep,
release readiness, and repository setup.

| Checklist | Purpose |
| --- | --- |
| `repository-ai-readiness-checklist.md` | Confirm a project has minimum AI-agent context. |
| `pre-implementation-checklist.md` | Check task clarity before code changes. |
| `code-review-checklist.md` | Review correctness, maintainability, and safety. |
| `owasp-review-checklist.md` | Review common OWASP security areas. |
| `pre-commit-checklist.md` | Check staged changes before committing. |
| `pull-request-checklist.md` | Ensure PR contains evidence and QA handover. |
| `release-readiness-checklist.md` | Confirm release risk, validation, and rollback. |
