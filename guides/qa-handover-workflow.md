# QA Handover Workflow

## Use This Prompt

Needs prompt: create a dedicated QA handover prompt if teams use this workflow often.
Until then, use this guide as the checklist and include the items below in the final handover.

QA handover should explain what changed, how to test it, what risks exist, and what evidence is available.

Include:

- scope;
- affected modules;
- acceptance criteria;
- test data;
- verification commands/results;
- known limitations;
- regression focus;
- security/performance/config notes.
