# QA Handover Checklist

Feature/bug/PR: Needs confirmation
Developer owner: Needs confirmation
QA owner: Needs confirmation
Environment/build/commit: Needs confirmation

## Scope

- [ ] Change summary is clear.
- [ ] In-scope behavior is listed.
- [ ] Out-of-scope behavior is listed.
- [ ] Related issue/requirement/PR is linked or marked Needs confirmation.

## Changed Areas

- [ ] Affected modules/pages/APIs/jobs/config are listed.
- [ ] API contract changes are described or marked N/A.
- [ ] Database/migration changes are described or marked N/A.
- [ ] Environment/config changes are described or marked N/A.

## Developer Verification Evidence

- [ ] Commands run and results are recorded.
- [ ] Manual checks are described.
- [ ] Screenshots/recordings/API examples/logs are attached when relevant.
- [ ] Checks not run are listed with reasons.

## QA Focus

- [ ] Primary scenarios to test are listed.
- [ ] Edge cases/risk areas are listed.
- [ ] Regression areas are listed.
- [ ] Test data/account needs are documented or marked Needs confirmation.

## Risk and Release

- [ ] Known limitations are listed.
- [ ] Security-sensitive areas are flagged.
- [ ] Performance/compatibility concerns are flagged.
- [ ] Rollback/deployment notes are linked or marked N/A.

## QA Readiness Decision

- [ ] Ready for QA.
- [ ] Ready with gaps listed below.
- [ ] Blocked.
